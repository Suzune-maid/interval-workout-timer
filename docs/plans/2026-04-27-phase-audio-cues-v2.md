# Phase Audio Cue V2 Plan

> **For Hermes:** 重新編排每段開始前的音訊時間軸：先播階段說明，再播開始音效，之後才開始倒數；每段倒數結束時再播結束音效。

## Goal
讓今天課表的每一段都遵循一致節奏：`階段說明 → 開始音效 → 倒數 → 結束音效`。

## Key Decisions
- 倒數要等階段說明與開始音效都播完才開始。
- 倒數結束時先播結束音效，再切到下一段的說明。
- 暫停後重新開始時，如果仍在同一段中途，不重播完整階段說明，只補播開始音效再恢復倒數。
- 開始／切段前的等待期視為 `preparing` 狀態，避免使用者在語音播放期間誤以為已開始倒數。
- 開始與結束音效使用可重用 WAV 素材，放在 `audio/fx/`。

## TDD Scope
1. 先為音訊播放器加測試，驗證：
   - 旁白會等到播放結束才算完成
   - `phase intro` 的順序是旁白（可選）→ 開始音效
   - `phase end` 會播結束音效
2. 再修改 `app.js`，讓倒數流程改成以 phase 為單位啟動，而不是先開倒數再播旁白。
3. 產生並提交開始／結束音效素材與索引檔。
4. 本機瀏覽器驗證時間軸與正式站 smoke test。

- 正式站資產與互動更新成功
